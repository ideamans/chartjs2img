# AI / docs artifact policy

chartjs2img uses a single-source-of-truth (SSOT) model for everything
agents and docs consume. The rules below apply unconditionally.

## Never hand-edit these files

They are regenerated by scripts; any manual edit will be overwritten:

- `docs/public/llms.txt`
- `docs/public/llms-full.txt`
- `docs/public/examples/**` (PNG + JSON renders, produced by `bun run docs:examples`)

The `rules` array inside `context7.json` is also a generated-in-spirit
concern â€” edit only after reading the "Why" below. Other fields in
`context7.json` (`projectTitle`, `folders`, `excludeFolders`, `excludeFiles`)
are hand-maintained.

## If you want to change what agents see, edit the source

| Want to changeâ€¦                               | Edit this                                                         |
| --------------------------------------------- | ----------------------------------------------------------------- |
| Chart.js + plugin option reference            | `src/llm-docs/<module>.ts` (each plugin has its own file)         |
| LLM reference registration / ordering         | `src/llm-docs/index.ts`                                            |
| CLI flags / usage banner                      | `src/index.ts` (`printUsage()` string)                             |
| HTTP endpoint shapes                          | `src/server.ts`                                                    |
| Renderer / cache / semaphore behavior         | `src/renderer.ts`, `src/cache.ts`, `src/semaphore.ts`              |
| Chart.js plugin bundle (versions, new plugin) | `src/template.ts` (`LIBS` map)                                     |
| Built-in example charts                       | `src/examples.ts` (`EXAMPLES` array)                               |
| Human-facing guide content (EN)               | `docs/en/**` (Japanese mirror at `docs/ja/**` â€” translate if you update)|
| Version number                                | `package.json.version` (propagates via `src/version.ts`)           |
| Plugin SKILL.md bodies                        | `plugins/chartjs2img/skills/*/SKILL.md`                            |
| Plugin manifest version                       | `plugins/chartjs2img/.claude-plugin/plugin.json` (keep in sync with `package.json.version`) |

After any of those edits, regenerate with `/regen-ai` (see the
regen-ai skill) or run `bun run ai:regen` directly.

## Why

Duplicated content drifts. The CLI help, the docs CLI reference, the
LLM bundle, the llms.txt link list, the SKILL.md examples â€” all need
the same information; keeping parallel copies by hand was the failure
mode in earlier iterations. Now the source is in one place (the code,
`src/llm-docs/`, `src/examples.ts`, `docs/en/`) and everything else is
derived.
